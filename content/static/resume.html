Summary

I am a Platform Engineer looking for challenging projects that have an impact on people.

I worked in the video-game industry for a cumulative time of 2 years, shipping two AAA games and one independent game. I am since then working on improving the Web Platform with Mozilla.

I developed a keen interest in Web Standards and designing APIs for the web and would like to keep working on making the Web a better place.

I like to write clean, simple and tested code. My main programming language is C++ but I have professional experiences with C, Java, Python, Objective-C, Bash, and Javascript/HTML/CSS. However, I believe that a language is a tool and a good software engineer should be able to switch from one to another easily.

Work Experience

  • 2010 - Present Senior Platform Engineer at Mozilla

    I am working on new APIs for the Web Platform, mostly to support the Firefox OS project and to make the Web competitive against native on Mobile. Involved in most API designs for Firefox OS, I have implemented a few of them in Firefox for Android and pushed some of them to standardization bodies. I also took part of the design of the security model of Firefox OS and implemented key parts of it.

    As a reviewer for the HAL and HTML/DOM modules and a Mozilla super-reviewer, I do a lot of code reviews to approve patches to land in mozilla-central and super-reviews to validate API changes.

    I am also involved in the W3C specification process as a co-chair of the SysApps working group and an active member of the WHATWG, HTML, DAP and WebApps working groups. I am editor of the Battery Status API, the Screen Orientation API, the Runtime and Security Model for Web Applications and the Network Information API.

    I implemented HTML5 Forms in Firefox with new features like constraint validation for forms, new HTML elements and new <input> types. I also mentored a few contributors to forms.

  • 2009 Google Summer of Code
    Google Summer of Code is a Google sponsored 3-month contribution to a FLOSS project.

    The goal of my Summer of Code was to implement a Gentoo Portage backend for PackageKit, an abstraction layer on top of package managers for Linux distributions. In addition of implementing the backend, this work involved extending Portage APIs, fixing bugs and adding features to Portage and PackageKit.

  • 2007 - 2009 Gameplay Programmer/Platform Engineer at Ubisoft

    While at university, I have done a few interships and short term contracts at Ubisoft, working on various games at various positions.

    Rayman Raving Rabbids 2 (Nintendo Wii) - 8 months
    I joined the team when the project was starting and left to get back to school after the code freeze. I started by prototyping some mini-games, and then was responsible for a regular mini-game and all six FPS mini-games. As the only Gameplay Programmer on these, I wrote the code handling the game mechanisms such as AI code to handle all sorts of enemies.

    Ghost Recon: Future Soldier (XBox 360/PS3) - 5 months
    I improved re-usability, maintenance and quality of a core part of a 10-year old game engine written in C/C++ with a short-term goal of reducing memory usage. I started an effort to have the code covered by a test suite.

    Undisclosed project - 3 months
    I was responsible for experimenting gameplay ideas for an experimental music-related game. This task required a lot of interaction with the Game Designer and the ability to quickly implement ideas and improve them iteratively by try/fix iterations.

  • 2006 Platform Engineer at Dancing Dots Studio (5-month internship)

    I wrote a 2D/3D sound engine based on OpenAL and a GUI engine based on CEGUI for the PC game Ride!. I also wrote the prototype of the rhythm game mode.
